Cacique API

A Cacique API é uma API desenvolvida pela Ethos com o conceito Serverless, executada na AWS através de uma função Lambda, que é requisitada pelo ERP Bling via webhook.

Esta API foi desenvolvida para enviar uma mensagem via WhatsApp sempre que um pedido for enviado no ERP Bling.

O que é Serverless?

Serverless é um modelo de computação em nuvem onde o provedor de nuvem (AWS, Google Cloud, Azure, etc.) é responsável por executar um pedaço de código ao responder a eventos. Isto significa que, em vez de manter um servidor funcionando o tempo todo, o servidor é iniciado apenas quando necessário, trazendo vantagens como economia de custos, escalabilidade automática e facilidade de implantação.

O que é AWS?

AWS (Amazon Web Services) é uma plataforma de serviços de nuvem que oferece poder computacional, armazenamento de banco de dados, entrega de conteúdo e outras funcionalidades para ajudar as empresas a escalar e crescer.

O que é Webhook?

Webhooks são uma forma útil de apps se comunicarem entre si automaticamente. Eles permitem que você envie dados de um aplicativo para outro sempre que um determinado evento ocorre. No caso da Cacique API, quando um evento específico ocorre no ERP Bling, um webhook é disparado para a função Lambda na AWS.

Sobre a API

A Cacique API foi desenvolvida com TypeScript no Node.js versão 16. Ela envia uma solicitação POST para um endpoint do Duotalk para que uma mensagem seja postada via WhatsApp, sempre que o status de pedido de um cliente da Tribalismo for "enviado".


Segurança

A API utiliza padrões de segurança via token para garantir a segurança dos dados, seguindo os processos mais avançados de engenharia de software desde a elicitação do requisito, design da solução, implementação/codificação, testes e validação/verificação.

CI/CD

A API usa princípios de Integração Contínua / Entrega Contínua (CI/CD), permitindo que mudanças de código sejam implementadas e testadas automaticamente, reduzindo o tempo de lançamento e aumentando a eficiência do desenvolvimento.

Escalabilidade

O código foi projetado pensando na escalabilidade, facilitando a inclusão de novos endpoints e funcionalidades.
